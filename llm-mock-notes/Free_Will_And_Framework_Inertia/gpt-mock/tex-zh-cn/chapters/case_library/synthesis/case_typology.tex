\section{案例类型学}

这四个案例旨在构成一个小型分析库：
\begin{enumerate}[nosep]
  \item \textbf{重新进入失败：} 目标状态此前明明已经激活过，但重新进入的成本太高。
  \item \textbf{边界失败：} 系统错过了唯一有用的转变区间，随后被旧路线重新捕获。
  \item \textbf{相邻盆地失败：} 努力并非不存在，但它被困在一个相邻状态里；那个状态看起来很有生产性，却阻断了真正目标。
  \item \textbf{替代状态失败：} 行动者试图停止旧机制，却没有构造出一个稳定的新机制。
\end{enumerate}

这种类型学的收益在于迁移。读者一旦理解了类型，就不再需要死记具体轶事；案例会变成一种可复用的诊断模式。
