# 数学与学术追求

## 身份认同
过去、现在和未来，我一直都是数学系的学生。没有人说数学系的学生不能编程、写代码或开发软件。

## 形式化证明与 Lean
这是一个哲学反思。在 Lean 证明中，最终目标是填补定理中的 `sorry` 缺口。因此，在证明过程中，虽然一些中间引理可能很难，但一个 `exact?` 往往可以通过库 API 解决它们，我们不需要关心 `exact?` 选择的 API 背后的底层推理。同样，一些琐碎的细节可能会被一个 `aesop` 或 `omega` 清除。所以我的结论是：当我们主要关心实现目标时，不必过分执着于细节；相反，将目标分解为子步骤，并设法消除每个 `sorry`。某个特定的 `sorry` 具体是如何解决的可能并不重要。同样，当人类指导 Agent 时，Agent 的结果对人类来说可能是不透明的；只要它有效，我们就可以接受。这符合我之前的“即时响应思维”：除非问题必须现在解决，否则我们就应该暂时绕过它。学海无涯；知识是无限的，而工作是有限的——我们不可能无限制地挖掘到底。

说到这里，其实回到了我学习 Lean 的初衷，即形式化数学证明，使其绝对严谨。从这个角度来看，我认为形式化那些初等数学问题，例如初等数论、实分析和线性代数中的一些简单证明，更符合我的个人兴趣。

## 数学基础
我认为当务之急是迫切提高我们的数学基础。虽然 AI 可以辅助我们编写代码，但它永远无法将数学定义灌输到我们的脑海中。
我的意思是，这个行业的一线从业者应该拥有受人尊敬的学术地位和丰厚的经济回报。因为这是用英语写的，下一部分我还没完全想好。我相信这在未来会成为现实，因为 AI4M 显然具有实用性和极高的价值。我也说不准——也许我应该咨询几位真正了解这一领域的教授。

这是一个普通的公式，不仅如此，我想了解它更深层的方面。

## 概率论
我认为概率论和随机过程确实非常有用。研究随机过程的人实际上是在快速加深对世界的深刻理解。

## 学习计划
虽然如果我现在开始深入研究第11章——本质上是第12章的数学基础——会暂时阻碍我彻底探索第12章，但我实际上发现第12章相当容易，所以不必过于担心。总而言之，先吃早餐然后再回来学习一会儿似乎是合理的。
