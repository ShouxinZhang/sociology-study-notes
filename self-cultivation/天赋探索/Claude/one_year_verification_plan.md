# 软件开发能力一年验证计划

> 基于天赋分析 v1 & v2，验证核心命题：  
> **"我能否把想象力、问题意识、跨领域联想、系统构思，通过软件原型稳定地表达出来？"**

---

## 一、验证目标的精确定义

### 1.1 "软件开发能力"对你来说意味着什么

你要验证的**不是**"能否成为传统意义上的强工程师"——那条路要求的是高频训练后快速自动化的执行型天赋，恰好是你天赋分析里的短板区。

你要验证的是：

> **能否用软件作为载体，把脑中的概念组合、世界观构造、跨领域联想，变成可运行、可展示、可迭代的现实产物。**

具体来说，这意味着：
- 你不需要手写红黑树，但需要能用现成工具和框架把一个系统搭起来
- 你不需要刷 LeetCode 到 hard，但需要能读懂别人的代码并在其上改造
- 你不需要从零实现 LLM，但需要能把 LLM API 编排成你想要的行为系统
- 你的核心竞争力不在代码本身，而在于**"用代码表达出别人想不到的东西"**

### 1.2 一年后的判定标准

| 级别 | 判定标准 | 具体表现 |
|------|---------|---------|
| **最低可接受** | 能独立完成一个中等复杂度的软件项目，从构思到可运行 demo | 至少有 1 个可展示的完整项目（有 README、能运行、别人能理解你在做什么）；能用 Python/TS 实现自己的想法而不是只停留在描述层面 |
| **合格** | 能产出 2-3 个展示天赋组合的项目，且至少一个有外部反馈 | 项目不仅能运行，而且体现了跨领域联想或世界观构造的独特性；在 GitHub/社区/朋友圈获得过"这个想法有意思"的反馈 |
| **超预期** | 形成了可持续的"想法→原型→迭代"工作流，且产出节奏稳定 | 3 个以上完整项目；其中至少 1 个进入迭代阶段（v2 以上）；开始有人主动询问你的项目或表示想参与；自己对"下一步做什么"有清晰规划而非茫然 |

### 1.3 什么信号意味着"需要调整方向"

- 连续 3 个月，每次坐下来写代码都感到纯粹的痛苦而非"有挑战但想继续"
- 完成项目后没有丝毫成就感，只觉得"终于结束了"
- 发现自己真正兴奋的部分始终只在构思阶段，一旦进入实现就彻底失去动力，且 AI 辅助也无法缓解
- 一年结束时，没有任何一个项目是你愿意向别人主动展示的

## 二、按季度拆分的执行计划

### Q1（月 1-3）：基础建设期 —— 从"能跑通"到"能表达"

**核心目标**：建立最小可用的技术工具箱，完成 2 个小型练手项目。

#### 月 1：搭建开发环境与语言感觉

**技术栈选择理由**：
- **Python**（主力）：生态最丰富，LLM/Agent 库最多，上手门槛最低，适合"工具借力"风格
- **TypeScript + React/Next.js**（辅助）：做前端展示用，让你的想法能被别人看到
- **Git + GitHub**：版本管理和作品展示窗口

**具体行动**：
- [ ] 用 Cursor/Copilot 辅助，完成 Python 官方教程的核心章节（函数、类、模块、文件 I/O、异常处理）
- [ ] 学会使用 `pip`/`uv` 管理依赖，`venv` 管理环境
- [ ] 跟着一个 Next.js 入门教程做出一个能部署的页面（Vercel 一键部署）
- [ ] 每天写 30 分钟代码，不追求质量，追求"手指习惯键盘"

**产出物**：
1. 一个个人 GitHub 仓库，包含学习笔记和代码片段
2. 一个部署在 Vercel 上的个人页面（哪怕只是 Hello World + 自我介绍）

#### 月 2：API 调用与数据流

**具体行动**：
- [ ] 学会调用 OpenAI / Claude / 本地 LLM 的 API（用 Python `httpx` 或官方 SDK）
- [ ] 理解 prompt engineering 基础：系统提示词、few-shot、structured output
- [ ] 学会用 JSON 做数据序列化，用 SQLite 做简单持久化
- [ ] 完成**练手项目 1**（见下方项目序列）

**产出物**：
1. 练手项目 1 的可运行代码 + README
2. 一份"LLM API 使用笔记"（记录踩过的坑）

#### 月 3：系统串联与第一次完整交付

**具体行动**：
- [ ] 学会用 FastAPI 或 Flask 搭一个简单后端
- [ ] 学会前后端联调的基本模式（前端 fetch → 后端 API → 返回 JSON）
- [ ] 完成**练手项目 2**（见下方项目序列）
- [ ] 回顾 Q1：哪些环节顺畅，哪些环节卡壳

**产出物**：
1. 练手项目 2 的可运行 demo + README
2. Q1 回顾文档（1 页，手写，不用长）

---

### Q2（月 4-6）：原型期 —— 做出第一个展示天赋组合的作品

**核心目标**：完成 1 个中等复杂度的项目原型，体现"想象力 + 跨领域联想 + 软件实现"的结合。

#### 月 4：原型设计与技术选型

**具体行动**：
- [ ] 从项目序列中选一个中等难度项目（推荐项目 3 或 4），花 1 周做概念设计
- [ ] 画出系统架构图（不需要完美，用 Mermaid 或手绘都行）
- [ ] 确定技术栈：哪些用现成库，哪些需要自己写
- [ ] 搭建项目骨架：目录结构、核心模块占位、基本数据模型

**产出物**：
1. 项目概念设计文档（1-2 页：做什么、为什么做、架构草图）
2. 项目骨架代码（能跑，但功能是空的）

#### 月 5：核心功能实现

**具体行动**：
- [ ] 集中实现项目最核心的 1-2 个功能模块
- [ ] 每周至少有一个可运行的中间版本（哪怕很粗糙）
- [ ] 遇到技术障碍时，优先用 AI 辅助解决，记录解决过程
- [ ] 开始学习 Agent 框架基础（LangChain / LangGraph / AutoGen / CrewAI，选一个入门）

**产出物**：
1. 核心功能可运行的 alpha 版本
2. 一份"开发过程中遇到的问题与解法"记录

#### 月 6：完善与第一次展示

**具体行动**：
- [ ] 补充 UI 或交互界面（Streamlit / Gradio / 简单前端）
- [ ] 写一份清晰的 README：是什么、为什么做、怎么跑、截图/录屏
- [ ] 把项目分享给至少 2 个人看，收集反馈
- [ ] Q2 回顾：原型做出来了吗？体现天赋组合了吗？哪里最兴奋，哪里最痛苦？

**产出物**：
1. 完整的项目原型（有 UI、有 README、能演示）
2. 至少 2 条外部反馈记录
3. Q2 回顾文档

---

### Q3（月 7-9）：迭代深化期 —— 从粗糙到有深度

**核心目标**：在 Q2 原型基础上深化，或启动一个更复杂的新项目；开始形成个人技术风格。

#### 月 7：技术能力升级

**具体行动**：
- [ ] 根据 Q2 经验，补充短板技术（可能包括：异步编程、数据库进阶、部署运维）
- [ ] 深入学习一个 Agent/Multi-Agent 框架，理解其架构设计
- [ ] 开始阅读 1-2 个开源项目的源码（选和你方向相关的，如 AutoGPT、BabyAGI、Generative Agents 等）

**产出物**：
1. 一份"开源项目阅读笔记"（记录架构、设计决策、可借鉴之处）
2. 一个技术短板补充的学习记录

#### 月 8-9：深化项目

**具体行动**：
- [ ] 选择路线 A 或 B：
  - **路线 A**：在 Q2 原型基础上做 v2（加入更复杂的交互、更丰富的 Agent 行为、更完整的世界观）
  - **路线 B**：启动项目序列中更高难度的新项目（推荐项目 5 或 6）
- [ ] 引入至少一个新技术元素（如：多 Agent 协作、知识图谱、向量数据库、实时交互）
- [ ] 保持每周至少一个可运行版本的节奏

**产出物**：
1. 深化后的项目 v2 或新项目的可运行版本
2. 一篇"我在做什么以及为什么"的文章（发在 GitHub README / 个人博客 / 知乎，任选）
3. Q3 回顾文档

---

### Q4（月 10-12）：整合展示期 —— 产出完整作品，回答核心问题

**核心目标**：产出至少 1 个完整度较高的代表作品；回答"软件开发是否是我天赋组合的有效载体"。

#### 月 10-11：打磨代表作

**具体行动**：
- [ ] 选择你迄今最满意的项目，投入 2 个月做深度打磨
- [ ] 目标：从"能用"到"好用"——交互体验、视觉呈现、边界情况处理
- [ ] 为项目写一份完整文档：设计理念、技术架构、使用指南、未来规划
- [ ] 尝试把项目部署到公开环境（GitHub Pages / Vercel / 自己的服务器）

**产出物**：
1. 代表作品 v3+（有完整文档、可公开访问）
2. 项目的 1 分钟演示视频或 GIF

#### 月 12：回顾与验证结论

**具体行动**：
- [ ] 回顾 12 个月的全部产出，列一个清单
- [ ] 对照"最低可接受 / 合格 / 超预期"标准，诚实评估
- [ ] 回答以下核心问题：
  - 软件开发是否让我的天赋组合找到了出口？
  - 我在哪些环节体验到了"心流"？
  - 我的瓶颈是技术能力不够，还是想法不够，还是其他？
  - 如果再给我一年，我想继续还是想转向？
- [ ] 根据结论，制定下一年规划（继续深入 / 调整方向 / 组合新策略）

**产出物**：
1. 12 个月产出总清单
2. 验证结论文档（2-3 页）
3. 下一年方向规划草案

## 四、关键检查点与调整机制

### 4.1 每月自检问题清单

每月最后一天，花 30 分钟回答以下问题（建议手写或语音记录）：

1. **这个月我写了多少行有意义的代码？**（不要求精确，感觉大概即可）
2. **这个月有没有一个时刻，我因为写代码而兴奋？** 如果有，是什么触发的？
3. **这个月遇到的最大技术障碍是什么？** 我是怎么解决（或没解决）的？
4. **这个月的产出物完成了吗？** 如果没有，差在哪里？
5. **我对"用软件表达想法"这件事的信心，比上个月更高还是更低了？**
6. **下个月我最想做的一件事是什么？**

### 4.2 季度落后时的缩小范围策略

如果某个季度结束时严重落后（完成度 < 50%），不要放弃，用以下策略缩小范围：

| 情况 | 调整策略 |
|------|---------|
| 技术学不动，卡在基础上 | 把项目复杂度砍半——功能减少到最核心的 1 个；允许用更多 no-code/low-code 工具（Streamlit、Gradio、v0.dev）先把东西做出来 |
| 时间不够，其他事太多 | 把月度目标改为双周目标；每个项目的完成标准从"可展示"降为"自己能跑通"；保底维持每周写代码的习惯（哪怕只有 1 小时） |
| 做着做着失去兴趣 | 换一个项目主题（从项目序列里重新选），但保持同样的技术练习目标；问自己是对"编程"失去兴趣还是对"这个项目"失去兴趣 |
| AI 辅助也搞不定 | 把问题拆到最小单元发到 Stack Overflow / GitHub Discussion / 技术社群提问；允许暂时跳过这个技术点，用其他方式绕过 |

### 4.3 方向调整的信号

以下任何一条如果在 **连续 2 个月** 内成立，就应该认真考虑调整方向：

- ❌ 你完全不期待下一次打开编辑器的时刻
- ❌ 你只在"不得不做"的时候才写代码，主动时间为零
- ❌ 完成一个项目后没有任何成就感，只有疲惫
- ❌ 你发现自己真正兴奋的事全部发生在代码之外（写作、画画、聊天、策划）
- ❌ 你用了大量 AI 辅助后仍然觉得"我完全无法理解这些代码在做什么"

如果出现以上信号，不要解读为"我失败了"，而是：

> 你成功排除了一个方向，现在有更好的信息来选择下一个载体。  
> 可能的替代载体：写作输出、内容策划、理论研究、人 + AI 混合创作。

## 五、推荐项目序列

以下项目由简到难排列，结合你的兴趣（Agent 系统、sandbox、世界观构造、LLM 应用）和实际水平设计。

---

### 项目 1：多人格 LLM 聊天室（入门）

**描述**：创建一个命令行/Streamlit 聊天器，内置 3-5 个不同性格的 AI 角色（如：逻辑分析师、诗人、毒舌评论家、哲学家），用户输入一个话题，所有角色依次回应，形成一场虚拟圆桌讨论。

**预计耗时**：2-3 周

**核心技术点**：
- LLM API 调用（OpenAI / Claude SDK）
- Prompt engineering（系统角色设定、温度调节）
- Python 基础（函数、循环、字符串处理）
- Streamlit 基础界面

**天赋组合体现**：
- **想象力**：设计角色性格和互动规则
- **世界观构造**：为聊天室设定场景背景（太空站里的圆桌会议？中世纪城堡的辩论？）
- **跨领域联想**：让不同"人格"代表不同学科视角

---

### 项目 2：故事分支生成器（入门+）

**描述**：输入一个故事开头，AI 生成 2-3 个可能的下一步，用户选择一个方向后继续生成，形成一棵交互式故事树。所有选择和路径保存在本地，可以回溯浏览。

**预计耗时**：3-4 周

**核心技术点**：
- LLM API + structured output（JSON 格式返回选项）
- 树形数据结构（用 dict/JSON 表示故事树）
- 文件 I/O（SQLite 或 JSON 文件持久化）
- 简单 Web UI（Next.js 或 Streamlit）

**天赋组合体现**：
- **想象力 + 叙事能力**：设计故事生成的 prompt 策略
- **结构感**：管理故事分支的数据结构
- **工具借力**：让 LLM 做"写作引擎"，自己做"架构师"

---

### 项目 3：Agent 社交模拟器（中等）

**描述**：创建 5-10 个有不同性格、记忆、目标的 AI Agent，让它们在一个虚拟空间中自主互动（聊天、结盟、争论、传播信息）。观察涌现行为。灵感来源：Stanford Generative Agents 论文。

**预计耗时**：6-8 周

**核心技术点**：
- Agent 框架（LangGraph / AutoGen / 自定义）
- 记忆系统（短期记忆 + 长期记忆摘要）
- 向量数据库基础（Chroma / FAISS，用于记忆检索）
- 事件循环系统（Agent 之间的消息传递）
- 可视化（前端展示 Agent 关系网络和对话记录）

**天赋组合体现**：
- **世界观构造**：设计 Agent 的社交世界规则
- **跨领域联想**：社会学 + AI + 涌现理论
- **问题发现**：观察"Agent 社会中会自发出现什么现象"
- **系统构思**：设计记忆、动机、交互的整体架构

---

### 项目 4：跨学科知识图谱探索器（中等）

**描述**：把你感兴趣的多个领域（数学、物理、社会学、哲学、AI）的核心概念做成知识节点，用 LLM 自动发现和生成跨领域连接，构建一个可交互浏览的知识图谱。点击任意两个节点，AI 解释它们之间可能的联系。

**预计耗时**：5-7 周

**核心技术点**：
- 知识图谱建模（Neo4j / NetworkX）
- LLM 用于关系抽取和概念解释
- 图可视化（D3.js / Pyvis / 前端图库）
- FastAPI 后端

**天赋组合体现**：
- **跨领域联想**：这是这项天赋的直接产品化
- **抽象结构感**：把模糊的"这两个领域有关系"变成明确的图结构
- **问题发现**：图谱本身会启发新的跨领域问题

---

### 项目 5：世界观 Sandbox（中高）

**描述**：创建一个可配置的虚拟世界生成器——用户定义物理规则、社会制度、资源分布、文明发展阶段等参数，系统自动生成一个虚拟世界的描述，并让 AI Agent 在其中模拟文明发展。世界可以保存、分享、fork。

**预计耗时**：8-10 周

**核心技术点**：
- 参数化世界生成系统
- Multi-Agent 模拟（多个 Agent 代表不同文明/势力）
- 时间线推进机制（回合制模拟）
- 状态持久化（数据库存储世界快照）
- 前端仪表盘（展示世界状态、文明进展、关键事件）

**天赋组合体现**：
- **世界观构造**：这就是你"构造世界观"天赋的直接载体
- **想象力**：设计世界参数和演化规则
- **系统构思**：抽象出"世界"的可配置模型
- **跨领域联想**：物理学 + 社会学 + 博弈论 + AI

---

### 项目 6：概念遗传算法实验平台（中高）

**描述**：把小说/故事/概念拆解为"原子模块"（角色特质、情节原型、世界观参数、冲突模式），然后用遗传算法式的组合和变异，让 LLM 评估和演化出"有趣度最高"的新概念组合。这直接对应你之前做过的"将不同小说原子化、模块化、自由重组"的想法。

**预计耗时**：6-8 周

**核心技术点**：
- 概念原子化的数据结构设计
- 遗传算法基础（选择、交叉、变异）
- LLM 做"适应度评估器"（评价概念组合的有趣程度）
- 迭代演化与可视化（每代最佳概念展示）

**天赋组合体现**：
- **想象力 + 结构感**：把创意过程系统化
- **数学发散**：遗传算法 + 概念空间的数学建模
- **跨领域联想**：进化论 + 叙事学 + 计算创意

### 项目 7：个人 Agent 工作台（高）

**描述**：为自己构建一个 Agent 系统，能够：接收你的日常想法/笔记/灵感，自动分类归档，发现跨笔记的联系，定期生成"你最近的想法图谱"，并在你需要时充当"思维伙伴"进行结构化对话。本质上是把你的天赋组合中的"知识库代替机械记忆"这一习惯产品化。

**预计耗时**：持续迭代项目，初版 8-10 周

**核心技术点**：
- RAG（检索增强生成）系统
- 向量数据库 + Embedding
- Multi-Agent 架构（归档 Agent、联想 Agent、对话 Agent）
- Telegram/Discord Bot 或 Web 前端
- 定时任务（定期生成想法图谱）

**天赋组合体现**：
- **自主探索 + 工具借力**：为自己的认知习惯定制工具
- **跨领域联想**：Agent 系统自动发现跨领域连接
- **问题发现**：系统帮你发现"你自己都没注意到的思维模式"

### 项目 8：多元宇宙叙事引擎（终极目标项目）

**描述**：这是你"多元小说宇宙自由 sandbox game"的软件化实现——一个平台，用户可以创建自己的宇宙设定，AI Agent 在其中扮演角色推动故事发展，多个用户的宇宙可以产生交叉和碰撞，故事线随参与者的选择动态分叉与合流。

**预计耗时**：这是一年后的方向性目标，一年内做出概念验证即可

**核心技术点**：前述所有技术的综合运用

**天赋组合体现**：这个项目本身就是你全部天赋组合的终极表达载体

## 八、最终提醒
